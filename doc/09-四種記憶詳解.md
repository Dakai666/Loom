# 四種記憶詳解

---

## 1. Episodic Memory（情景記憶）

### 用途

記錄當前 session 中依序發生的事件。相當於 agent 的「工作記憶」——只屬於這個 session，session 結束後會被壓縮寫入 Semantic Memory。

### 何時寫入

每次 tool call 的開始與結束都會寫入：

```python
# Tool 開始
await episodic.add(EpisodicEntry(
    event="tool_call",
    tool="read_file",
    arguments={"path": "doc/SUMMARY.md"},
    session_id=self.session_id,
))

# Tool 結果
await episodic.add(EpisodicEntry(
    event="tool_result",
    tool="read_file",
    result="4869 chars",  # 截斷至 SMAL
    duration=0.12,
    session_id=self.session_id,
))
```

### 資料結構

```python
@dataclass
class EpisodicEntry:
    id: int | None
    session_id: str
    event: str              # "tool_call" | "tool_result" | "error"
    tool: str | None        # 工具名稱（event=tool_* 時）
    arguments: str | None   # 參數摘要（截斷）
    result: str | None      # 結果摘要（SMAL 截斷）
    duration: float | None  # 耗時（秒）
    created_at: str         # ISO timestamp
```

### Session 結束時的處理

```
Session 結束
    ↓
所有 EpisodicEntry 送入 LLM 提取 FACT
    ↓
每個 FACT upsert 至 SemanticMemory
    ↓
原始 EpisodicEntry 刪除（SQLite VACUUM reclaim 空間）
```

FACT 格式：
```
FACT: [session:abc123] 用戶要求建立 doc/ 目錄，已建立 doc/SUMMARY.md
```

## 2. Semantic Memory（語義記憶）

### 用途

儲存關於世界、程式碼庫、專案、協作者的事實。長期保存，可 upsert（後寫覆蓋先寫），可驗證失效。

### API

```python
# 寫入（upsert by key）
await semantic.upsert(
    key="project:db_schema",
    value="資料庫使用 SQLite，WAL 模式，路徑 ~/.loom/memory.db",
    confidence=0.9,
)

# 讀取
entry = await semantic.get("project:db_schema")
# → SemanticEntry(key="project:db_schema", value=..., confidence=0.9, ...)

# 刪除
await semantic.delete("project:db_schema")

# 列出最近
recent = await semantic.list_recent(limit=20)

# 搜尋
results = await semantic.search("SQLite WAL")
```

### Anti-pattern 寫入（v0.2.5.1 Counter-factual Reflection）

工具執行失敗時，LLM 反思產生的 Anti-pattern 以特殊 key 格式寫入：

```python
# Key 格式：skill:<tool_name>:anti_pattern:<timestamp>
await semantic.upsert(
    key=f"skill:{tool_name}:anti_pattern:{timestamp}",
    value="[Anti-pattern] 嘗試在 Windows 路徑使用 Unix 路徑分隔符導致 read_file 失敗；下次應先檢查 OS",
    confidence=0.8,
    source="reflection",
)
```

此類 key 可被 `recall("anti_pattern <tool>")` 召回，agent 在下次執行同類工具前會看到警告。

### 資料結構

```python
@dataclass
class SemanticEntry:
    id: int | None
    key: str
    value: str
    confidence: float           # 0.0 - 1.0，預設 0.8
    source: str | None          # "session:xxx" | "agent:xxx" | "manual" | "reflection"
    created_at: str
    updated_at: str
    embedding: str | None       # Phase 5：向量（JSON 字串）
```

### Phase 5：衝突追蹤（5F-1）

當 upsert 時發現 key 已存在且 value 不同：

```python
# 舊值寫入 metadata.history[]
entry.metadata["history"].append({
    "old_value": old_value,
    "source": old_source,
    "overwritten_at": old_updated_at,
})

# 保留最近 3 筆歷史
entry.metadata["history"] = entry.metadata["history"][-3:]
```

## 3. Procedural Memory（程序記憶）

### 用途

儲存「如何做」的技能——有效的 prompt patterns、成功的重構策略、經過驗證的工作流程。每個技能是一個 **Skill Genome**。

### Skill Genome 結構

```python
@dataclass
class SkillGenome:
    id: str                     # 唯一識別，"refactor_extract_function"
    version: int                # 版本號，每次更新 +1
    confidence: float           # EMA confidence，0.0-1.0
    usage_count: int            # 累積使用次數
    success_rate: float        # 歷史成功率
    parent_skill: str | None   # 父技能（用於技能繼承）
    deprecation_threshold: float = 0.3  # 低於此值自動廢棄
    tags: list[str]             # ["refactor", "python", "function"]
    body: str                   # 技能主體（有效時的 prompt 或描述）
    created_at: str
    updated_at: str
```

### EMA Confidence 更新

每次 Skill 執行後，結果回饋寫入，confidence 透過 EMA 更新：

```python
def record_outcome(self, success: bool):
    # EMA = α * new + (1 - α) * old
    # α = 0.3（新鮮度權重）
    alpha = 0.3
    self.confidence = alpha * (1.0 if success else 0.0) + (1 - alpha) * self.confidence
    self.usage_count += 1

    # 成功率同步更新
    self.success_rate = (
        (self.success_rate * (self.usage_count - 1) + (1.0 if success else 0.0))
        / self.usage_count
    )

    # 自動廢棄檢查
    if self.confidence <= self.deprecation_threshold:
        self.deprecate()
```

### 自動廢棄

```python
def deprecate(self):
    self.body = f"[DEPRECATED] {self.body}"
    self.metadata["deprecated"] = True
    self.metadata["deprecated_at"] = datetime.now().isoformat()
```

廢棄的技能仍存在於 DB 中，但 `list_active()` 預設不會返回。

### Skill Import Pipeline

外部技能透過 Lens + Pipeline 匯入：

```
外部 skill JSON
    ↓ Lens（HermesLens / OpenAIToolsLens）
SkillGenome 格式
    ↓ Pipeline（schema check + confidence gate + 去重）
upsert 至 ProceduralMemory
```

詳細：[32-Skill-Import.md](32-Skill-Import.md)

## 4. Relational Memory（關係記憶）

### 用途

儲存三元組（subject, predicate, object）形式的關係性事實：

```
subject,          predicate,    object
"user",           "prefers",   "concise answers"
"project:loom",   "uses",      "SQLite WAL"
"task:deploy",    "depends_on","task:build"
```

適合記錄用戶偏好、系統關係、任務依賴等。

### Anti-pattern 三元組（v0.2.5.1）

反事實反思產生的 Anti-pattern 也寫入 RelationalMemory：

```python
# Counter-factual Reflection 寫入的關係三元組
await relational.relate(
    subject=f"skill:{tool_name}",
    predicate="has_anti_pattern",
    object="使用 Unix 路徑分隔符處理 Windows 路徑",
    confidence=0.8,
)

await relational.relate(
    subject="loom-self",
    predicate="should_avoid:<tool_name>",
    object="在 Windows 環境下使用 / 作為路徑分隔符",
    confidence=0.9,
)
```

`MemoryIndex` 在 session 開始時讀取 `loom-self` 主詞的 `should_avoid` 三元組（Self-Portrait），讓 agent 知道自己過去應避免什麼。

### Self-Portrait 三元組（v0.2.5.3 SelfReflectionPlugin）

`loom-self` 主詞的觀察三元組由 SelfReflectionPlugin 寫入：

```python
# 三種 predicate 类型
(loom-self, tends_to, <observation>)      # 傾向性觀察
(loom-self, should_avoid, <observation>)    # 應避免行為
(loom-self, discovered, <observation>)       # 新發現
```

### API

```python
# 寫入三元組
await relational.relate(
    subject="user",
    predicate="prefers",
    object="繁體中文回答",
    confidence=1.0,
)

# 查詢（主詞）
prefs = await relational.query_relations(subject="user")
# → [("user", "prefers", "繁體中文回答")]

# 查詢（謂詞）
all_prefs = await relational.query_relations(predicate="prefers")
# → [("user", "prefers", "繁體中文回答"), ...]

# 查詢（主詞+謂詞）
answer_lang = await relational.query_relations(
    subject="user",
    predicate="prefers"
)
# → [("user", "prefers", "繁體中文回答")]

# 刪除
await relational.delete("user", "prefers", "繁體中文回答")
```

### 資料結構

```python
@dataclass
class RelationEntry:
    id: int | None
    subject: str
    predicate: str
    object: str
    confidence: float          # 0.0-1.0
    created_at: str
    updated_at: str
```

### 與工具整合

```python
# relate 工具（GUARDED）
@loom.tool(trust_level=TrustLevel.GUARDED)
async def relate(
    subject: str,
    predicate: str,
    obj: str,
    confidence: float = 1.0,
) -> str:
    """寫入關係到 Relational Memory"""
    await relational_memory.relate(subject, predicate, obj, confidence)
    return f"已記住：{subject} {predicate} {obj}"

# query_relations 工具（SAFE）
@loom.tool(trust_level=TrustLevel.SAFE)
async def query_relations(
    subject: str | None = None,
    predicate: str | None = None,
) -> str:
    """查詢關係記憶"""
    results = await relational_memory.query_relations(subject, predicate)
    return "\n".join(f"{s} {p} {o}" for s, p, o in results)
```

## 四種記憶的 Schema 對照

| 欄位 | Episodic | Semantic | Procedural | Relational |
|------|----------|----------|------------|------------|
| id | ✅ | ✅ | ✅ | ✅ |
| key/name | session_id+seq | key | id | — |
| content/value | event+tool+result | value | body | object |
| confidence | — | ✅ | ✅ | ✅ |
| source | ✅ | ✅ | — | — |
| timestamp | created_at | created/updated_at | created/updated_at | created/updated_at |
| parent | — | — | parent_skill | — |
| metadata | ✅ | ✅ | ✅ | — |
