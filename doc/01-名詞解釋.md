# 名詞解釋

本文檔收錄 Loom 框架中的核心術語，提供簡潔的定義與參照連結。

---

## A

### Agent
在 Loom 中，Agent 是執行任務的主體。它由 Prompt Stack 驅動，透過 Harness Layer 呼叫工具，產生的經驗存入 Memory Layer。

### AsyncOpenAI / AsyncAnthropic
Loom 使用的 LLM API client（來自 `openai` / `anthropic` 套件）。在 LoomSession 初始化時建立，session 內複用，不在每次請求時重新創建。

---

## B

### Barista
六種內建 Personality 之一。風格為「清晰 + 溫暖」，用比喻橋接複雜概念，交付時分段進行。參見 [28-Personalities.md](28-Personalities.md)。

### Blast Radius
工具執行後可能影響的範圍。Loom 的 `BlastRadiusMiddleware` 會在 GUARDED / CRITICAL 工具執行前評估影響範圍並請求用戶確認。

---

## C

### ConfirmFlow
處理需要用戶確認的通知流程。支援 APPROVED / DENIED / TIMEOUT 三種結果。參見 [25-ConfirmFlow.md](25-ConfirmFlow.md)。

### Context Budget
Loom 的 token 配額管理系統。當使用量達到設定閾值（預設 80%）時自動觸發 `_smart_compact()` 壓縮。參見 [15-Context-Budget.md](15-Context-Budget.md)。

### Cortex
見 Cognition Layer。

### CronTrigger
基於 cron 表達式（5-field）的時間觸發器。例如 `0 9 * * 1-5` 表示每週一至週五 09:00 執行。參見 [20-觸發器詳解.md](20-觸發器詳解.md)。

---

## D

### DAG（Directed Acyclic Graph）
有向無環圖。Loom 的 Task Engine 用 DAG 表示任務之間的依賴關係，確保有先後順序的工作不被並行執行。參見 [17-Task-Engine.md](17-Task-Engine.md)。

---

## E

### EMA（Exponential Moving Average）
指數移動平均。用於 Skill Genome 的 confidence 更新——每次 Skill 使用後，根據成功/失敗結果平滑更新 confidence 分數。參見 [10-Skill-Genome.md](10-Skill-Genome.md)。

### EventTrigger
事件觸發器。當系統內部 emit 特定事件時觸發（如 `error_rate_threshold`）。可用 `loom autonomy emit <event_name>` 手動觸發。參見 [20-觸發器詳解.md](20-觸發器詳解.md)。

### Episodic Memory
情景記憶。記錄本次 session 中依序發生的事件（每次 tool call 都會寫入一筆記錄）。Session 結束後會被壓縮寫入 Semantic Memory。參見 [09-四種記憶詳解.md](09-四種記憶詳解.md)。

---

## F

### Fact
從 Episodic Memory 壓縮後提取的事實片段。會寫入 Semantic Memory，長期保存。格式為 `FACT: <描述>`。

---

## H

### Harness Layer
工具生命週期的管理框架。是 Loom 的骨幹，負責 Middleware Pipeline、Tool Registry、Trust Hierarchy。參見 [04-Harness-概述.md](04-Harness-概述.md)。

---

## I

### Intent
意圖描述。在 loom.toml 中定義 Autonomy 觸發時的意圖文字，Action Planner 將其翻譯為實際的 prompt 交給 LLM 執行。

---

## L

### Lens
外部技能/工具的轉接器。將其他框架的 skill 或 tool schema 轉換為 Loom 的標準格式。參見 [30-Lens-系統.md](30-Lens-系統.md)。

### LoomSession
Loom 的核心 session 類。持有 LLM client、Memory 實例、Middleware Pipeline，負責串接整個請求流程。

---

## M

### Memory Index
記憶索引。一個極輕量的目錄結構，永久駐留在 system prompt 中，告訴 Agent「記憶庫裡有什麼」，讓 Agent 按需主動呼叫 `recall()` 召回。參見 [12-Memory-Index.md](12-Memory-Index.md)。

### Middleware
中介軟體。攔截每一次 tool call 的前後執行，提供日誌、追蹤、風險評估等橫切關懷。參見 [06-Middleware-詳解.md](06-Middleware-詳解.md)。

### Multi-turn
多輪對話。Loom 在單一 session 內維護完整的 message history，並在需要時自動壓縮以控制 Context Budget。

---

## N

### Notification
通知。Loom 對外發送的結構化訊息，包含類型（INFO / CONFIRM / INPUT / ALERT / REPORT）、內容、metadata。參見 [23-Notification-概述.md](23-Notification-概述.md)。

### NotificationRouter
通知路由器。將通知 fan-out 到所有已註冊的 Notifier 適配器（CLI / Webhook / Telegram / Discord），單一 channel 失敗不阻擋其他 channel。

---

## P

### Personality
人格。一種臨時疊加的認知濾鏡，影響 Agent 的思考方式與表達風格，但不改變核心身份（SOUL）。存放於 `personalities/*.md`。參見 [26-Prompt-Stack.md](26-Prompt-Stack.md)。

### Plugin
插件。Loom 的擴充單位，可同時註冊 tools / middleware / lenses / notifiers / session hooks。參見 [31-Plugin-系統.md](31-Plugin-系統.md)。

### Procedural Memory
程序記憶。存放「如何做」的技能（Skill Genome），具有版本號與 EMA confidence。參見 [09-四種記憶詳解.md](09-四種記憶詳解.md)。

### Prompt Stack
提示詞棧。三層結構：SOUL.md → Agent.md → Personality.md，後層疊加補充前層。參見 [26-Prompt-Stack.md](26-Prompt-Stack.md)。

---

## R

### Recall
召回。一個 SAFE 等級的工具，讓 Agent 按需從 Semantic Memory / Procedural Memory 中檢索相關內容，而非一次全部載入。參見 [12-Memory-Index.md](12-Memory-Index.md)。

### Reflexive
反思能力。Loom 的設計目標之一——Agent 能觀察並推理自己的執行路徑，透過 Reflection API 提供自我分析能力。

### Relational Memory
關係記憶。儲存三元組（subject, predicate, object），記錄偏好、關聯、限制等關係性事實。參見 [09-四種記憶詳解.md](09-四種記憶詳解.md)。

---

## S

### Semantic Memory
語義記憶。儲存關於世界 / codebase 的事實，以 key-value 形式長期保存，可驗證失效。參見 [09-四種記憶詳解.md](09-四種記憶詳解.md)。

### Skill Genome
技能的基因結構。一個帶有 confidence、使用次數、成功率的技能描述，支援 EMA 更新與自動廢棄。參見 [10-Skill-Genome.md](10-Skill-Genome.md)。

### SOUL.md
Loom 的核心身份定義檔。永駐 system prompt（約 300 tokens），描述身份、思維原則、與用戶的互動方式。參見 [27-SOUL-設計.md](27-SOUL-設計.md)。

### Sub-agent
子代理。透過 `spawn_agent` 啟動的獨立工作單元，與父 session 共享 Semantic Memory，有獨立的 session_id 寫入 Episodic。參見 Phase 5E 規格。

---

## T

### TaskGraph
任務圖。DAG 實現，支援 Kahn's topological sort 自動推導可並行節點。參見 [17-Task-Engine.md](17-Task-Engine.md)。

### TaskScheduler
任務排程器。基於 asyncio.gather，在同一 level 內並發執行 TaskGraph 中的可並行節點。參見 [18-Task-Scheduler.md](18-Task-Scheduler.md)。

### ToolRegistry
工具註冊表。Loom 的工具管理中心，儲存所有 ToolDefinition，並提供 `to_anthropic_schema()` / `to_openai_schema()` 輸出。參見 [07-Tool-Registry.md](07-Tool-Registry.md)。

### Trust Level
信任等級。三級（SAFE / GUARDED / CRITICAL）決定工具是否需要人類確認。參見 [05-Trust-Level.md](05-Trust-Level.md)。

---

## W

### WebhookNotifier
Webhook 通知適配器。透過 HTTP POST JSON 發送通知，並支援外部系統透過 `/webhook/reply` endpoint 回覆 CONFIRM / DENY。參見 [24-Notifier-適配器.md](24-Notifier-適配器.md)。

### WAL Mode
Write-Ahead Logging。SQLite 的 journal mode，Loom 的 Memory Layer 使用 WAL 模式提升併發讀取效能。
