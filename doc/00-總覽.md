# Loom 總覽

> *The loom is what the harness belongs to. Claude Code is one thread; Loom is the machine that weaves any thread into the same quality fabric.*

---

## Loom 是什麼？

**Loom** 是一個 **harness-first、memory-native、self-directing** 的 agent 框架。

它的目標不是取代任何現有 agent，而是成為一個能將任何優秀 agent 的設計精華吸收、並以統一品質運作的通用平台。

---

## 核心價值

| 價值 | 意義 |
|------|------|
| **Harness-first** | 工具生命週期是一等公民，不是功能附加物。每一個工具調用都經過 Middleware Pipeline 的審視與控制。 |
| **Memory-native** | 記憶是架構的基底（substrate），不是後期加上的插件。四種記憶類型從一開始就存在於核心設計中。 |
| **Reflexive** | Agent 能觀察並推理自己的執行路徑。透過 Reflection API，系統可以分析自己的行為並持續改進。 |
| **Absorbent** | 框架本身能學習其他框架的優點並整合進來。Lens 系統讓外部工具、Skill、Prompt Pattern 能無縫匯入。 |
| **Self-directing** | 可在無人類觸發的情況下自主決策並行動。Autonomy Engine 讓 Loom 能根據時間、事件或條件自動執行。 |

## 設計信條

> **「工具是公民，不是附屬品。」**

Loom 不把工具當作「功能」來對待。當一個工具被註冊進 Loom，它會獲得：
- 完整的生命週期管理（註冊 → 路由 → 執行 → 審計 → 卸載）
- 明確的信任級別（SAFE / GUARDED / CRITICAL）
- Middleware 的裝飾與增強（日誌、追蹤、風險評估）
- 寫入 Semantic Memory 的機會（讓每次執行都成為學習素材）

## 誰適合用 Loom？

✅ **你正在開發一個需要嚴謹工具管理的 agent**  
✅ **你需要持久化、多層次的記憶系統**  
✅ **你的 agent 需要在沒有人類盯著的情況下自主行動**  
✅ **你希望框架能從每次執行中學習並改進**  
✅ **你想用同一套框架管理多個不同的 LLM provider**

❌ **你只需要一個簡單的 chat wrapper**  
❌ **你不關心工具的信任層級與審計**

## 快速開始

```bash
# 啟動互動式 session
loom chat

# 啟動 TUI 介面
loom chat --tui

# 查看記憶狀態
loom memory list

# 啟動自主引擎
loom autonomy start
```
