# 新增工具

本指南說明如何在 Loom 中新增自訂工具。

---

## 工具結構

Loom 的工具定義分兩層：

1. **`ToolDefinition`**（資料類）：工具的元資料，包含名稱、描述、trust_level、capabilities、schema
2. **`executor` 函數**（協程）：實際的工具邏輯

**不建議**繼承 `ToolDefinition` 或建立自訂 dataclass——直接使用 `@loom.tool` decorator 最簡潔。

---

## 方式 1：`@loom.tool` decorator（推薦）

`@loom.tool` 自動從函數簽名推斷參數 schema、從 docstring 提取 description、包裝為 `executor` 函數並註冊進 `AdapterRegistry`：

```python
# loom/extensibility/adapter.py
@loom.tool(trust_level="guarded", description="部署指定的服務")
async def deploy(call: ToolCall) -> ToolResult:
    """
    Deploy a service to the configured target environment.

    Args:
        service_name: Name of the service to deploy.
        environment: Target environment (default: staging).
    """
    service = call.args.get("service_name")
    env = call.args.get("environment", "staging")

    if not service:
        return ToolResult(
            call_id=call.id,
            tool_name="deploy",
            success=False,
            error="service_name is required",
            failure_type="validation_error",
        )

    try:
        result = await _do_deploy(service, env)
        return ToolResult(call_id=call.id, tool_name="deploy", success=True, output=result)
    except Exception as exc:
        return ToolResult(
            call_id=call.id,
            tool_name="deploy",
            success=False,
            error=str(exc),
            failure_type="execution_error",
        )
```

`@loom.tool` 會自動：
- 解析函數參數名稱與類型Hint → JSON Schema `properties`
- 識別 required 參數（無預設值的參數）
- 從 docstring 第一行取 description
- 將函數包裝為 `ToolDefinition(executor=fn, ...)` 並寫入 `AdapterRegistry`

## 方式 2：直接建立 ToolDefinition

對於需要明確控制 schema 或 capabilities 的場景，直接建立 `ToolDefinition`：

```python
# loom/core/harness/registry.py
from .registry import ToolDefinition, ToolCapability
from .permissions import TrustLevel

tool_def = ToolDefinition(
    name="my_tool",
    description="我的自訂工具",
    trust_level=TrustLevel.GUARDED,
    capabilities=ToolCapability.MUTATES,
    parameters={
        "type": "object",
        "properties": {
            "param1": {"type": "string", "description": "必要參數"},
            "param2": {"type": "string", "description": "可選參數", "default": "default_value"},
        },
        "required": ["param1"],
    },
)
```

## ToolCall 結構

工具執行函數接收一個 `ToolCall`：

```python
@dataclass
class ToolCall:
    id: str
    tool_name: str
    args: dict[str, Any]
    trust_level: TrustLevel
    session_id: str
    timestamp: datetime
    metadata: dict[str, Any]
```

---

## ToolResult 結構

工具回傳 `ToolResult`，**不拋异常**：

```python
@dataclass
class ToolResult:
    call_id: str
    tool_name: str
    success: bool
    output: Any = None
    error: str | None = None
    failure_type: str | None = None   # validation_error / permission_denied / timeout / execution_error / model_error
    duration_ms: float = 0.0
    metadata: dict[str, Any]
```

---

## Trust Level 選擇

| 等級 | 值 | 使用時機 |
|------|-----|---------|
| SAFE | `TrustLevel.SAFE` | 唯讀、不修改任何狀態 |
| GUARDED | `TrustLevel.GUARDED` | 有副作用（寫檔、網路請求）|
| CRITICAL | `TrustLevel.CRITICAL` | 不可逆或跨系統破壞性操作 |

---

## Capability 標記

| Capability | 值 | 使用時機 |
|-----------|-----|---------|
| `MUTATES` | `ToolCapability.MUTATES` | 修改檔案、記憶、資料庫 |
| `EXEC` | `ToolCapability.EXEC` | 執行 shell / 子程序（每次重新確認）|
| `NETWORK` | `ToolCapability.NETWORK` | 發出對外 HTTP 請求 |
| `AGENT_SPAN` | `ToolCapability.AGENT_SPAN` | 啟動子代理（每次重新確認）|

## 完整範例：Calculator Tool

```python
# my_tools/calculator.py
import ast
import operator
from loom.extensibility import loom
from loom.core.harness.middleware import ToolCall, ToolResult

@loom.tool(trust_level="safe", description="執行數學計算")
async def calculator(call: ToolCall) -> ToolResult:
    """
    Safely evaluate a mathematical expression.

    Args:
        expression: A mathematical expression such as "2 + 3 * 4"
    """
    expr = call.args.get("expression", "")

    allowed_ops = {
        ast.Add: operator.add,
        ast.Sub: operator.sub,
        ast.Mult: operator.mul,
        ast.Div: operator.truediv,
        ast.Pow: operator.pow,
    }

    try:
        tree = ast.parse(expr, mode="eval")
        result = _eval_node(tree.body)
        return ToolResult(call_id=call.id, tool_name="calculator", success=True, output=str(result))
    except Exception as exc:
        return ToolResult(
            call_id=call.id,
            tool_name="calculator",
            success=False,
            error=f"Evaluation error: {exc}",
            failure_type="execution_error",
        )

def _eval_node(node):
    if isinstance(node, ast.Constant):
        return node.value
    elif isinstance(node, ast.BinOp):
        return allowed_ops[type(node.op)](_eval_node(node.left), _eval_node(node.right))
    else:
        raise ValueError(f"Unsupported node: {ast.dump(node)}")
```

## 將工具加入 Plugin

```python
# my_plugin/plugin.py
from loom.extensibility import LoomPlugin
from . import calculator  # 觸發 @loom.tool decorator

class MyPlugin(LoomPlugin):
    name = "my_plugin"
    version = "1.0"
    # tools() 自動發現所有 @loom.tool 裝飾過的函數
```

Plugin 放入 `~/.loom/plugins/my_plugin/` 後，Loom 會自動發現並載入。

---

## 總結

1. 首選 `@loom.tool` decorator（自動 schema 推斷）
2. 複雜 schema 使用 `ToolDefinition` + 手寫 `parameters` dict
3. 工具函數簽名接收 `ToolCall`，回傳 `ToolResult`（不拋异常）
4. 選擇適當的 `TrustLevel` 和 `ToolCapability`
5. Plugin 放入 `~/.loom/plugins/<name>/` 自動發現
