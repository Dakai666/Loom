# 測試指南

本指南說明 Loom 框架的測試方式。

---

## 測試框架

Loom 使用 pytest 作為測試框架：

```bash
# 安裝測試依賴
pip install pytest pytest-asyncio pytest-cov

# 執行所有測試
pytest

# 執行並顯示覆蓋率
pytest --cov=loom --cov-report=html
```

## 單元測試

### 範例：測試 Tool

```python
# tests/unit/test_harness/test_tool.py
import pytest
from loom.core.harness.tool import Tool, ToolResult

class DummyTool(Tool):
    name = "dummy"
    description = "A dummy tool for testing"
    trust_level = "SAFE"
    
    async def execute(self, args: dict, context: dict) -> ToolResult:
        return ToolResult(success=True, output=args.get("input"))

@pytest.fixture
def tool():
    return DummyTool()

@pytest.mark.asyncio
async def test_tool_execute(tool):
    result = await tool.execute({"input": "test"}, {})
    
    assert result.success is True
    assert result.output == "test"

@pytest.mark.asyncio
async def test_tool_error(tool):
    # Tool 不會拋出異常，錯誤通過 ToolResult 表示
    result = await tool.execute({}, {})
    
    assert result.success is True
    assert result.output is None
```

### 範例：測試 Memory Store

```python
# tests/unit/test_memory/test_store.py
import pytest
import tempfile
from pathlib import Path
from loom.core.memory.store import MemoryStore

@pytest.fixture
async def store():
    """創建測試用的記憶儲存"""
    with tempfile.TemporaryDirectory() as tmp:
        db_path = Path(tmp) / "test.db"
        store = MemoryStore(db_path)
        await store.init()
        yield store
        await store.close()

@pytest.mark.asyncio
async def test_write_semantic(store):
    await store.write_semantic("test_key", "test_value", {"source": "test"})
    
    result = await store.get_semantic("test_key")
    
    assert result is not None
    assert result.value == "test_value"

@pytest.mark.asyncio
async def test_search_semantic(store):
    # 寫入多條記錄
    await store.write_semantic("apple", "fruit", {})
    await store.write_semantic("banana", "fruit", {})
    await store.write_semantic("carrot", "vegetable", {})
    
    # 搜尋
    results = await store.search_semantic("fruit", limit=10)
    
    assert len(results) == 2
    assert all("fruit" in r.value.lower() for r in results)
```

## 整合測試

### 範例：測試 Chat Session

```python
# tests/integration/test_chat_session.py
import pytest
from loom.core.session.manager import SessionManager
from loom.core.harness.executor import HarnessExecutor

@pytest.mark.asyncio
async def test_session_lifecycle():
    # 創建 session
    manager = SessionManager()
    session = await manager.create(personality="architect")
    
    assert session.status == SessionStatus.ACTIVE
    assert session.message_count == 0
    
    # 加入訊息
    await session.add_message(Message(role="user", content="Hello"))
    await session.add_message(Message(role="assistant", content="Hi!"))
    
    assert session.message_count == 2
    
    # 完成 session
    await manager.complete(session.id)
    
    updated = await manager.get(session.id)
    assert updated.status == SessionStatus.COMPLETED
```

### 範例：測試 Tool 執行

```python
# tests/integration/test_tool_execution.py
import pytest
from loom.core.harness.executor import HarnessExecutor
from loom.core.harness.registry import ToolRegistry

@pytest.mark.asyncio
async def test_tool_execution_with_middleware():
    registry = ToolRegistry()
    harness = HarnessExecutor(registry)
    
    # 註冊測試工具
    harness.registry.register(MockTool())
    
    # 加入 middleware（記錄所有呼叫）
    calls = []
    async def logging_middleware(tool, args, next_fn):
        calls.append({"tool": tool.name, "args": args})
        return await next_fn()
    
    harness.add_middleware(logging_middleware)
    
    # 執行
    result = await harness.execute("mock_tool", {"input": "test"})
    
    assert result.success is True
    assert len(calls) == 1
    assert calls[0]["tool"] == "mock_tool"
```

## Mock 技巧

### 範例：Mock LLM 回應

```python
# tests/fixtures/conftest.py
import pytest
from unittest.mock import AsyncMock

@pytest.fixture
def mock_llm():
    """Mock LLM 回應"""
    llm = AsyncMock()
    llm.complete.return_value = "Mocked response"
    return llm
```

# tests/unit/test_cognition/test_router.py

## 執行測試

```bash
# 執行所有測試
pytest

# 執行特定目錄
pytest tests/unit/test_memory/

# 執行特定檔案
pytest tests/unit/test_harness/test_tool.py

# 執行特定測試
pytest tests/unit/test_harness/test_tool.py::test_tool_execute

# 顯示 print 輸出
pytest -s

# 停在第一個失敗
pytest -x

# 只執行上次失敗的測試
pytest --lf
```

## 覆蓋率

```bash
# 生成覆蓋率報告
pytest --cov=loom --cov-report=term-missing

# 生成 HTML 報告
pytest --cov=loom --cov-report=html
open htmlcov/index.html

# 最小覆蓋率檢查
pytest --cov=loom --cov-fail-under=80
```

## CI/CD 整合

```yaml
# .github/workflows/test.yml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.11'
    
    - name: Install dependencies
      run: |
        pip install -e ".[test]"
    
    - name: Run tests
      run: pytest --cov=loom --cov-fail-under=80
    
    - name: Upload coverage
      uses: codecov/codecov-action@v3
```

## 總結

| 命令 | 功能 |
|------|------|
| `pytest` | 執行所有測試 |
| `pytest tests/unit/` | 執行單元測試 |
| `pytest -s` | 顯示 print 輸出 |
| `pytest -x` | 停在第一個失敗 |
| `pytest --lf` | 只執行上次失敗 |
| `pytest --cov=loom` | 生成覆蓋率報告 |
| `pytest --cov-fail-under=80` | 檢查最小覆蓋率 |
